"每次只记那一组的空间，而且经常显示 0" — 用户感知的数据失真问题需要在本次彻底修复。
文件大小获取 → 累计逻辑 → 持久化存储 数据流

src/services/stats-service.tstotalFreedBytes += freedBytes 持久化到 AsyncStorage
  • 删除后设置页"释放空间"数值增加，误差 < 20%
  • 重启 App 后数值保持累计值，不归零
  • 多次删除连续累加，不跳变

02

照片地理位置信息展示

P2

浏览照片时在日期下方显示拍摄城市名，数据来源为照片 EXIF 中的 GPS 坐标。通过 getAssetInfoAsync(id) 读取 info.location，用 Nominatim API 逆向地理编码到城市级，API 不可用时降级显示原始坐标。无 GPS 数据的照片不显示地点行。

验收标准

  • 有 GPS 的照片显示 "📍 城市名"
  • 无 GPS 的照片不显示地点行
  • 切换照片时地点实时更新
  • API 失败时降级显示坐标
