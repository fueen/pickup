> 版本：v1.4 | 日期：2026-05-24 | 状态：已批准

## 概述

首页重构（相册选择 → 二级照片浏览）、Hub 柱状图、Review 页按钮增强、释放空间 Bug 修复、Tab 图标色修复。共 7 项。

## REQ-01: 首页重构

### 路由变更

```
Tab 导航:
  index.tsx     → 重写为相册选择首页（3列 grid）
  hub.tsx       → 不变（REQ-03 重写内容）
  settings.tsx  → 不变（REQ-05 修复逻辑）

非 Tab 路由:
  browse.tsx    → 新建，承载原 index.tsx 照片浏览逻辑
  review.tsx    → 修改（REQ-04/06）
  paywall.tsx   → 不变
```

### 数据流

```
index.tsx:
  useEffect → MediaLibrary.getAlbumsAsync() → Album[]
  Album[] 映射为 3列 FlatList/ScrollView grid
  每格: 封面缩略图 + 相册名 + 照片数
  onPress → router.push({ pathname: '/browse', params: { albumId, albumTitle } })

browse.tsx:
  useLocalSearchParams → { albumId, albumTitle }
  albumId === '__all__' → 加载全部照片（原有逻辑）
  albumId !== '__all__' → getAssetsAsync({ album: albumId, first: N, ... })
  其他逻辑（滑动、标记、删除）复用现有 usePhotoEngine
  headerLeft: 返回箭头 → router.back()
```

### 组件

- **AlbumGrid** (index.tsx 内联或新建): FlatList numColumns=3, renderItem 渲染 AlbumCard
- **AlbumCard**: 封面 Image + 名称 Text + 计数 Text
- **browse.tsx**: 迁移原 index.tsx，新增 albumId 参数读取 + 返回按钮

## REQ-03: Hub 月度柱状图

### 组件

**MonthlyChart.tsx** (新建 `src/components/hub/`):
- 纯 RN View 实现，无第三方依赖
- 接口: `{ data: { month: string; count: number }[] }`

### 数据逻辑

hub.tsx useEffect:
1. `MediaLibrary.getAssetsAsync({ first: 1 })` 获取总数
2. 分批拉取所有照片的 `creationTime`（复用分页逻辑）
3. 按月聚合: `{ "2026-01": 42, "2026-02": 38, ... }`
4. 取最近 12 个月，传入 MonthlyChart

### 视觉

- 柱子: 黄色渐变 (`#FFCC00` → `#CCA300`)，圆角顶部，等宽
- 当前月份高亮（更亮色 or 边框）
- X 轴: 月份简称 (1月、2月...)
- 数字标注在柱顶
- 支持横向 ScrollView（月份 > 6 时）

## REQ-05: 释放空间 0B 修复

### 根因推测

`stats-service.ts` 的 `recordDeleted` 中，删除前获取文件大小的逻辑在 SDK 54 下可能失效（旧版 `FileSystem.getInfoAsync` 已弃用）。

### 修复

delete-service.ts 删除流程:
1. 删除前: `const file = new File(asset.uri); const { size } = await file.statAsync()` (SDK 54 新版 API)
2. 或: `MediaLibrary.getAssetInfoAsync(asset).then(info => info.fileSize)`
3. 累加 size → `statsService.recordDeleted(totalBytes)`
4. StatsContext 确保从 AsyncStorage 正确读取 `freedSpace`
