### Task 2: Restore Service Tests and Implementation

**Files:**
- Create: `__tests__/unit/delete-service-restore.test.ts`
- Modify: `src/services/delete-service.ts`
- Modify: `jest.setup.js`

- missing cached records are skipped and reported as failures
- partial failure restores the successes and leaves failures.

- [ ] **Step 2: Run tests to verify RED**

Run: `npx jest __tests__/unit/delete-service-restore.test.ts --runInBand`

Expected: FAIL because `restoreDeletedPhotos` does not exist.

- [ ] **Step 3: Implement restoreDeletedPhotos**

Add `restoreDeletedPhotos(records)` to `src/services/delete-service.ts`.

Return shape:

```ts
export interface RestoreResult {
  successCount: number;
  failedCount: number;
  restoredIds: string[];
  failedIds: string[];
  errors: string[];
}
```

Use `new File(record.uri).exists` as the local cache guard, then `await MediaLibrary.createAssetAsync(record.uri)`. Remove only restored IDs from recent delete storage.
